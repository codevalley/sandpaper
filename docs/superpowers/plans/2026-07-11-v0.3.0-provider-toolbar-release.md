# Sandpaper v0.3.0 Provider Toolbar and Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make provider choice, independent history/reset, diagnostics, and usage truthful in the floating toolbar, then reconcile documentation and qualify v0.3.0 for release.

**Architecture:** The server injects one validated provider bootstrap and accepts provider IDs with each turn. Global lifecycle remains single-turn while browser selection is tab/session-local. Provider-tagged frames prevent transcript contamination, and scoped reset/default endpoints coordinate browser convenience state with server-owned local state.

**Tech Stack:** Node.js HTTP/SSE, static browser JavaScript/CSS, sessionStorage, Node test runner, Playwright Chromium, static HTML documentation, npm/GitHub release workflow.

## Global Constraints

- Consume the reviewed runtime interfaces exactly; do not recreate provider/session/preference logic in browser code.
- The server validates provider identity and attaches it to every accepted turn, frame, and replay record.
- Preserve one global active turn across pages, tabs, and providers.
- Never silently fall back to another provider.
- Treat diagnostic `available` as the server-owned ready predicate: installed, compatible, and authenticated.
- Browser selection is session-local; `Make default` is the only action that mutates the manifest.
- Transcripts are keyed by project, page, and provider; cross-provider hidden context is never transferred.
- `New session` clears only one page/provider and clears browser history only after server success.
- Actual selected-document bytes remain the only Saved/Replied and Undo authority.
- Show Claude cost only when supplied and Codex tokens only when supplied; never estimate missing values.
- Preserve existing toolbar error recovery, direct-edit rollback, accessibility, reduced motion, CSS isolation, and narrow usability.
- Historical Claude-specific evidence remains unchanged; current-state documentation becomes dual-provider.
- Release remains blocked until both providers pass install, edit, resume, switch, reset, and package smoke.

```js
var bootstrapData = JSON.parse(bootstrap.getAttribute('data-sandpaper-bootstrap') || '{}');
var providerKey = 'sp-provider:v1:' + bootstrapData.projectId;
var providers = bootstrapData.providers || [];
var selectedProvider = bootstrapData.initialProvider;
try { selectedProvider = sessionStorage.getItem(providerKey) || selectedProvider; } catch (error) {}

function selectProvider(id) {
  var provider = providers.find(function (entry) { return entry.id === id; });
  if (!provider || !provider.available || busy) return false;
  selectedProvider = id;
  try { sessionStorage.setItem(providerKey, id); } catch (error) {}
  syncProviderPresentation(provider);
  rehydrateTranscript();
  return true;
}
```

```js
function transcriptKey(providerId) {
  return ['sp-thread:v2', bootstrapData.projectId, location.pathname, providerId].join(':');
}

function persist() {
  try { sessionStorage.setItem(transcriptKey(selectedProvider), thread.innerHTML); } catch (error) {}
}

function rehydrateTranscript() {
  var saved = '';
  try { saved = sessionStorage.getItem(transcriptKey(selectedProvider)) || ''; } catch (error) {}
  thread.innerHTML = saved;
  rebuildTurnIndexForProvider(selectedProvider);
}
```

Every turn record stores its accepted provider.
