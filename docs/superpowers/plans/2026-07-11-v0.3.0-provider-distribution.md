# Sandpaper v0.3.0 Provider Distribution Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Install, upgrade, diagnose, and package one shared Sandpaper workflow for both Claude Code and Codex without clobbering user-owned agent configuration.

**Architecture:** Provider-neutral workflow references are the canonical action source. Thin Claude command wrappers and one Codex `$sandpaper` dispatcher install into namespaced trees, while focused helpers own manifest migration, managed instruction blocks, hook merging, lifecycle preservation, and diagnostics.

**Tech Stack:** Node.js 18+ ESM, JSON and Markdown files, Claude `.claude` configuration, Codex `.agents` skills and `.codex/hooks.json`, Node test runner, npm pack, zero runtime dependencies.

## Global Constraints

- Install both integrations by default; permit explicit Claude-only or Codex-only installation.
- Keep runtime provider preference separate from installed integrations.
- Preserve all bytes outside Sandpaper-owned namespaces and managed blocks.
- Project Codex hooks require explicit trust; installation must never claim they are active merely because configuration exists.
- Both providers read and stamp the same repository and brain; provider session IDs never enter the brain.
- Existing v1 manifests migrate to Claude default without losing project, counters, theme, port, source, or brain identity.
- Upgrade and rebuild preserve default provider, integrations, hook intent, theme, identity, and counters.
- `doctor` reports saved authentication method but never reads or prints credentials.
- Provider diagnostic `available` means ready to run: installed, compatible, and authenticated.
- Preserve zero runtime dependencies and Node.js `>=18`.
- Published package allowlists remain exact and secret/path exclusions remain strict.

```js
export const MANIFEST_VERSION = 2;
export const PROVIDERS = ['claude', 'codex'];

export function migrateManifest(value) {
  if (!value || typeof value !== 'object' || Array.isArray(value)) throw new TypeError('Invalid manifest');
  if (value.version !== 1 && value.version !== 2) throw new Error(`Unsupported manifest version: ${value.version}`);
  return {
    ...value,
    version: MANIFEST_VERSION,
    defaultProvider: PROVIDERS.includes(value.defaultProvider) ? value.defaultProvider : 'claude',
    integrations: Array.isArray(value.integrations) && value.integrations.length
      ? [...new Set(value.integrations)] : [...PROVIDERS],
    hooksEnabled: value.hooksEnabled !== false,
  };
}
```

- [ ] **Step 3: Implement recursive namespaced installation and managed blocks**

```js
export function upsertManagedBlock(file, { begin, end, content }) {
  const source = existsSync(file) ? readFileSync(file, 'utf8') : '';
  const starts = source.split(begin).length - 1;
  const ends = source.split(end).length - 1;
  if (starts > 1 || ends > 1 || starts !== ends) return { ok: false, changed: false, error: 'Invalid Sandpaper markers' };
  const block = `${begin}\n${content.trim()}\n${end}`;
  const next = starts === 1
    ? source.slice(0, source.indexOf(begin)) + block + source.slice(source.indexOf(end) + end.length)
    : `${source}${source && !source.endsWith('\n') ? '\n' : ''}${block}\n`;
  if (next === source) return { ok: true, changed: false, action: 'unchanged' };
  mkdirSync(dirname(file), { recursive: true });
  writeFileSync(file, next);
  return { ok: true, changed: true, action: starts ? 'updated' : 'added' };
}
```

- [ ] **Step 3: Implement capability-based diagnostics and effective-option reuse**

`probeClaude` reuses the runtime diagnostic contract: run `claude --version`, then parse only `loggedIn` and the authentication method from `claude auth status --json`. It returns `available:true` only for structured authenticated status and never returns email, organization, subscription identity, or raw command output.

```js
export function probeCodex(runCommand) {
  const version = runCommand('codex', ['--version']);
  if (version.status !== 0) return { available: false, compatible: false, authMethod: null, unavailableCode: 'binary_missing' };
  const execHelp = runCommand('codex', ['exec', '--help']);
  const resumeHelp = runCommand('codex', ['exec', 'resume', '--help']);
  const login = runCommand('codex', ['login', 'status']);
  const text = `${execHelp.stdout}\n${resumeHelp.stdout}`;
  const compatible = ['--json','--ignore-user-config','--ignore-rules'].every((flag) => text.includes(flag));
  const authMethod = /ChatGPT/i.test(login.stdout) ? 'chatgpt' : /API key/i.test(login.stdout) ? 'api-key' : login.status === 0 ? 'unknown' : null;
  return { available: compatible && authMethod !== null, compatible, authMethod, version: version.stdout.trim(), unavailableCode: compatible && authMethod ? null : compatible ? 'unauthenticated' : 'incompatible' };
}
```

`upgrade` and `rebuild` read effective manifest options first and pass them explicitly to integration/hook/scaffold helpers. `rebuild` backs up only `brain/`; it must not delegate to an optionless `installSkill`. `doctor` merges brain inspection with installation inspection and classifies missing unselected providers as warnings, missing selected providers as errors, and Codex trust as an external warning with `/hooks` guidance.
