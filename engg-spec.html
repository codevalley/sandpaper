Sandpaper

Engineering spec — the current Claude-only runtime.

A local zero-runtime-dependency Node.js bridge that lets you refine a Claude-generated HTML document in the browser. The server owns authentication, lifecycle, snapshots, hashes, undo, and reload truth; Claude remains the only production execution provider. The runner seam is provider-neutral internally so Codex can arrive in v0.3.0 without another server rewrite.

The invariant

The HTML file on disk is the document. Claude and Hands mutate those bytes; the server verifies before/after hashes and reloads from disk. Model stdout supplies frames, never document HTML.

01
Overview

What ships in the stabilized runtime

Run node bin/cli.js <document.html|directory>. The server binds to 127.0.0.1, injects the toolbar at response time, and keeps the file on disk authoritative.

Current behavior. Sand runs scoped Claude turns; Hands edits rich text, moves or deletes sibling blocks, and provides one-level undo. The server reports Saved only after a real byte change and otherwise reports Replied. Every rejected optimistic edit restores the exact prior DOM.

Honest scope. The production UI runs Claude Code only. Screenshot paste, branching, durable regenerated-element anchoring, and first-class Codex execution are planned, not shipped.

Node ≥18 built-ins only at runtime. Playwright is development-only and excluded from the package files.

03
The bridge

Claude is the current runner; the server owns truth

Production injects runTurn() into the server-owned lifecycle. It is spawned with cwd = docDir so Claude's directory-scoped resume lookup remains reliable:

claude -p <prompt> \
  --output-format stream-json --verbose --include-partial-messages \
  --permission-mode acceptEdits \
  --allowedTools Read,Edit,Write,MultiEdit \
  --append-system-prompt <CONTRACT> \
  [--resume <sessionId>]

The Claude runner maps NDJSON into normalized status, assistant-delta, and edit-summary frames. It does not assign turn IDs, decide Saved/Replied, expose undo, or schedule reloads; those are server decisions backed by actual bytes and snapshots.

Session. Claude's session id is stored under .sandpaper/session.json, which the shared path policy never serves. Losing it does not lose the artifact: the file remains the source of truth.

The deterministic fake runner covers reply-only, changed, unchanged, delayed, malformed, and failed execution without invoking Claude in CI. Waiting remains a reserved UI state.
04
The page protocol

Authenticated HTTP, replayable SSE, and serialized writes

method · pathpurpose
GET /<page>Serve an allowed repository HTML file with a fresh process token injected into the toolbar bootstrap; never write that token to disk.
GET /__sandpaper/eventsAuthenticated SSE. Replays the current status, then page-scoped status/reload frames for the tab's client ID.
POST /__sandpaper/turnReserve lifecycle before body read; validate authenticated JSON and scope; snapshot/hash; return 202 {turnId} or a structured error.
POST /__sandpaper/write · /domApply direct rich-text, sibling move, or delete operations with a direct-undo snapshot; reject conflicts with an active turn.
POST /__sandpaper/undo · /undo-directRestore an existing unconsumed snapshot only when no conflicting turn is active.
GET /__sandpaper/toolbar.{js,css}Serve the overlay assets from the package's own public/.
GET /<path>Serve only paths accepted by the shared lexical + canonical repository policy.

Anatomy of one turn

  1. Pick optionally records a data-cid, selector, and text snippet; Sand and Hands remain mutually exclusive.
  2. The client sends process token, stable per-tab client ID, same-origin JSON, prompt, page, and optional scope.
  3. The server reserves receiving before body read, validates the request/page/scope, then snapshots and hashes the page.
  4. The server enters running and invokes the current Claude runner; normalized frames stream over SSE.
  5. Claude writes with file tools. The watcher records the page/client source and routes reloads without hiding other-tab changes.
  6. A terminal runner frame triggers a new hash. Changed bytes plus a retained snapshot expose Saved/Undo; unchanged bytes expose Replied.
  7. Errors remain terminal and visible. The next accepted turn starts a new lifecycle; every client path restores usable UI after rejection.
Direct edits are optimistic in the initiating tab only. Server confirmation or exact rollback closes the transaction; other relevant tabs receive page-scoped reloads from disk.
05
Security model

Local-process protection plus one shared repository boundary

controldetail
Loopback bindServer listens on 127.0.0.1 only — never 0.0.0.0. Not exposed to the network.
Process tokenA cryptographically random in-memory token is injected at response time; mutation routes and SSE require it.
Browser originMutation requests require valid loopback Host, same-origin evidence, a client ID, and the expected JSON content type.
Tool restriction--allowedTools Read,Edit,Write,MultiEdit — no Bash, no web fetch, no MCP. Claude can't run shell or open connections during a turn.
Repository policyAllow ordinary files and .github/**; deny .git, .sandpaper, unknown hidden namespaces, env/credential/key material, traversal, null bytes, and escaping symlinks.
Mutation policyAll hidden paths, invalid pages/CIDs/operations, conflicts, malformed JSON, and over-limit bodies fail with structured status/code/message JSON.

Boundary: this protects a single-user local process against cross-origin browser requests, DNS-rebinding-style access, accidental secret serving, and unsafe repository links. It is not hosted multi-user authentication.

doctor evaluates href, src, and data-ref with this same policy, so a blocked canonical link fails before publish instead of becoming an invisible runtime break.

Found & fixed

Verified failures now have regression contracts

The stabilization pass kept the original fixes and added end-to-end contracts around the boundaries that had been inferred rather than proven.

Security
path policy + local API — secret/runtime paths, unknown hidden namespaces, traversal, symlink escapes, foreign origins, invalid tokens/hosts/clients/content types, malformed JSON, and body overruns are rejected.
Lifecycle
server.js — request-body races, false idle after errors, false Saved outcomes, stale undo, and conflicting direct writes now have one server-owned state machine.
Reload
watch + SSE — current status is replayed; page/client identity replaces global suppression; other tabs see confirmed disk changes.
Browser
sp-client + toolbar — 4xx/5xx/malformed/network/runner errors restore the composer; Sand/Hands, undo, optimistic edits, focus, ARIA, and CSS isolation are verified in Chromium.
Brain
setup.js + brain.js — source/package drift, editorial mismatch, stale fallback counts, blocked references, and fake data-cid-only anchors are independently detected.

Node coverage: repository path policy; HTTP auth and structured errors; concurrency; current-state SSE; changed/unchanged/error outcomes; AI/direct undo; write/move/delete; snapshot pruning; scoped reloads; parser/edit/client helpers; scaffold and doctor truth checks.

Chromium coverage: toolbar injection; Saved/Replied; rejection recovery; Sand/Hands; direct transactions and rollback; undo; two-tab reload; timeline/shortcuts; focus/ARIA/reduced motion; host CSS isolation; brain derivation with atomic fetch failure fallback.

Release-only live step: run one disposable Claude edit and record it before the human confirms the v0.2.1 tag. Codex is not invoked because no Codex runner ships yet.

CI runs Node 18/20/22 and Chromium on Node 20. Playwright remains development-only; the published package keeps zero runtime dependencies.
