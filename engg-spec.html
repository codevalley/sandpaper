Sandpaper

Engineering spec — one truthful runtime for Claude Code and Codex.

A local zero-runtime-dependency Node.js bridge that lets you refine an HTML document with Claude Code or Codex in the browser. An internal static registry selects the adapter; the server owns authentication, one server-owned global turn lifecycle, snapshots, hashes, undo, and reload truth.

The invariant

The HTML file on disk is the document. Providers and Hands mutate those bytes; the server verifies before/after hashes and reloads from disk. Model stdout supplies frames, never document HTML.

01
Overview

What ships in the dual-provider runtime

Run node bin/cli.js <document.html|directory>. The server binds to 127.0.0.1, injects the toolbar at response time, and keeps the file on disk authoritative.

Current behavior. Sand runs explicitly selected Claude Code or Codex turns; Hands edits rich text, moves or deletes sibling blocks, and provides one-level undo. The server reports Saved only after a real byte change and otherwise reports Replied. Every rejected optimistic edit restores the exact prior DOM.

Entry points. Both integrations install by default and consume one workflow set and one brain: /sandpaper:<action> in Claude Code and $sandpaper <action> in Codex. Explicit solo installs remain supported. Screenshot paste, branching, and durable regenerated-element anchoring remain planned.

Node ≥18 built-ins only at runtime. Playwright is development-only and excluded from the package files.

03
The bridge

Two adapters; the server owns truth

Production dispatches exactly one validated registry entry's runTurn() inside the server lifecycle. Both children use cwd = docDir. Claude runs:

claude -p <prompt> \
  --output-format stream-json --verbose --include-partial-messages \
  --permission-mode acceptEdits \
  --allowedTools Read,Edit,Write,MultiEdit \
  --append-system-prompt <CONTRACT> \
  [--resume <sessionId>]

Codex reuses saved codex login authentication and runs a controlled embedded process:

codex --ask-for-approval never --sandbox workspace-write \
  --config web_search="disabled" \
  --config sandbox_workspace_write.network_access=false \
  --config project_doc_max_bytes=0 \
  --disable multi_agent --disable apps \
  exec --ignore-user-config --ignore-rules --json [resume <threadId>] <prompt>

The child environment removes CODEX_API_KEY and OPENAI_API_KEY overrides; it does not copy credentials. Saved Codex auth and the provider's own state directory remain accessible. These flags disable network, web search, apps, multi-agent behavior, user config, and repository rules, but do not claim OS-level isolation.

The adapters map NDJSON into normalized status, assistant-delta, edit, warning, and usage frames. Claude edit frames can carry hunks and supplied cost; Codex edit frames carry reported path/kind summaries and its completion can carry supplied total tokens. Neither value is estimated or converted. Adapters do not assign turn IDs, decide Saved/Replied, expose undo, or schedule reloads; those are server decisions backed by actual bytes and snapshots.

Session. .sandpaper/session.json stores opaque IDs by page and provider. A Claude ID can never reach Codex. Browser transcripts are separately keyed by project/page/provider and are noncanonical convenience state; the shared brain is durable truth.

The deterministic fake runner covers reply-only, changed, unchanged, delayed, malformed, and failed execution without invoking Claude in CI. Waiting remains a reserved UI state.
04
The page protocol

Authenticated provider protocol, replayable SSE, and serialized writes

Every served page receives bootstrap {projectId, initialProvider, defaultProvider, providers[]}. Provider diagnostics expose only available, compatible, authMethod, version, and unavailableCode beside ID and label.

method · pathpurpose
GET /<page>Serve an allowed repository HTML file with a fresh process token injected into the toolbar bootstrap; never write that token to disk.
GET /__sandpaper/eventsAuthenticated SSE. Replays the current status, then page-scoped status/reload frames for the tab's client ID.
POST /__sandpaper/turnAuthenticate and validate {prompt, page, provider}, provider readiness, page, and global idle state before reservation; snapshot/hash; return 202 {ok:true, turnId, provider} or a structured error.
POST /__sandpaper/provider-defaultAccept {provider}; require a registered available provider and global idle state; atomically update the local manifest; return exactly {ok:true, defaultProvider}.
POST /__sandpaper/session/resetAccept {page, provider}; require a registered provider, mutable canonical page, and global idle state; clear only that page/provider resume ID; return exactly {ok:true, page, provider}. Browser history clears only after this success.
POST /__sandpaper/write · /domApply direct rich-text, sibling move, or delete operations with a direct-undo snapshot; reject conflicts with an active turn.
POST /__sandpaper/undo · /undo-directRestore an existing unconsumed snapshot only when no conflicting turn is active.
GET /__sandpaper/toolbar.{js,css}Serve the overlay assets from the package's own public/.
GET /<path>Serve only paths accepted by the shared lexical + canonical repository policy.

All POST routes require the process token (X-Sandpaper-Token), stable per-tab X-Sandpaper-Client, loopback Host, same-origin evidence, JSON content type, and bounded request body. Global busy requests fail with 409 turn_in_progress, including default and reset requests from another page. Failures are structured {ok:false,error:{code,message}}.

Accepted output uses provider-tagged status, assistant_delta, edit, usage, and lifecycle frames. Turn frames include server turnId, validated provider, canonical page, and phase. Lifecycle is global; status/reload delivery remains page-aware.

Anatomy of one turn

  1. Pick optionally records a data-cid, selector, and text snippet; Sand and Hands remain mutually exclusive.
  2. The client sends process token, stable per-tab client ID, same-origin JSON, prompt, page, provider, and optional scope.
  3. The server performs provider/page/request validation before reservation, then snapshots and hashes the page.
  4. The server enters running and invokes the selected adapter; normalized provider-tagged frames stream over SSE.
  5. The provider writes with directory-level write ability. The watcher records page/client source and routes reloads without hiding other-tab changes.
  6. A terminal runner frame triggers a new hash. Changed bytes plus a retained snapshot expose Saved/Undo; unchanged bytes expose Replied.
  7. Errors remain terminal and visible. The next accepted turn starts a new lifecycle; every client path restores usable UI after rejection.
Direct edits are optimistic in the initiating tab only. Server confirmation or exact rollback closes the transaction; other relevant tabs receive page-scoped reloads from disk.
05
Security model

Local-process protection plus one shared repository boundary

controldetail
Loopback bindServer listens on 127.0.0.1 only — never 0.0.0.0. Not exposed to the network.
Process tokenA cryptographically random in-memory token is injected at response time; mutation routes and SSE require it.
Browser originMutation requests require valid loopback Host, same-origin evidence, a client ID, and the expected JSON content type.
Provider restrictionsClaude receives file-only tools. Controlled Codex disables network, web search, apps, multi-agent behavior, user config, and repository rules and strips API-key environment overrides.
Repository policyAllow ordinary files and .github/**; deny .git, .sandpaper, unknown hidden namespaces, env/credential/key material, traversal, null bytes, and escaping symlinks.
Mutation policyAll hidden paths, invalid pages/CIDs/operations, conflicts, malformed JSON, and over-limit bodies fail with structured status/code/message JSON.

Boundary: this protects a single-user local process against cross-origin browser requests, DNS-rebinding-style access, accidental secret serving, and unsafe repository links. It is not hosted multi-user authentication or an OS sandbox. Provider processes receive directory-level write ability. Sandpaper's selected-document bytes and snapshot are the only Saved/Replied/Undo authority. Provider-reported external paths are best-effort detection: Sandpaper cannot verify or undo them and does not auto-revert concurrent work.

doctor evaluates href, src, and data-ref with this same policy, so a blocked canonical link fails before publish instead of becoming an invisible runtime break.

07
Found & fixed

Verified failures now have regression contracts

The stabilization pass kept the original fixes and added end-to-end contracts around the boundaries that had been inferred rather than proven.

Security
path policy + local API — secret/runtime paths, unknown hidden namespaces, traversal, symlink escapes, foreign origins, invalid tokens/hosts/clients/content types, malformed JSON, and body overruns are rejected.
Lifecycle
server.js — request-body races, false idle after errors, false Saved outcomes, stale undo, and conflicting direct writes now have one server-owned state machine.
Reload
watch + SSE — current status is replayed; page/client identity replaces global suppression; other tabs see confirmed disk changes.
Browser
sp-client + toolbar — 4xx/5xx/malformed/network/runner errors restore the composer; Sand/Hands, undo, optimistic edits, focus, ARIA, and CSS isolation are verified in Chromium.
Brain
setup.js + brain.js — source/package drift, editorial mismatch, stale fallback counts, blocked references, and fake data-cid-only anchors are independently detected.
Historical defect prose remains in the append-only brain log and decision ledger. This section states the current regression boundary rather than replaying the original P0 audit.
