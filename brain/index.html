Board-first — the canvas is where explanations live

06-30 · board 005

The decision: when my reply would be a substantial summary or explanation, the board is the reply — it lands here, and the terminal gets one line (📋 … → on the canvas). Not written twice.

We dropped the Stop-hook “nag”. A hook can't tell what's board-worthy — it only sees length, so it over- or under-fires, and it never boards anything itself; it just pokes me, and the call is still mine. So the fix puts that judgment in the natural flow instead of bolting on a dumb detector. This board is the first written that way — a reply, not an afterthought.

Terminal · steershort answers · confirmations · the back-and-forth · the one-line pointer
Canvas · showsummaries · architectures · comparisons · walkthroughs — the rich stuff

Honest caveat: this is a default, not a hard lock — boarding a judgment can't be enforced like “you changed code, go stamp.” Your tripwire is the pointer: no pointer means nothing was boarded, so poke me.

Stamping keeps that settled layer current: after a substantial turn I prepend a one-line log row, rewrite the NOW focus, refresh the machine-readable digest, record any decision/gotcha, and flip the plan status. A Stop hook enforces it — change code without stamping and it blocks. The canvas is where thinking is formed; stamping is how the durable parts get kept.
