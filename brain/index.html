The brain links outside itself — on purpose

07-01 · board 006

Good instinct to question it — but the out-links aren't sloppiness, they're the spine. Sandpaper's one rule is link, never copy: the brain is a thin index over canonical truth (the spec, the source), never a second copy of it. Copy the content in and it drifts the moment the code changes — the exact rot the brain exists to kill. So a data-ref into ../src/server.js is the design working, not leaking.

Intra-brain linkscover ↔ lenses ↔ books — relative, self-contained, travel anywhere
Canonical refs→ the spec & the real source — meant to point outside brain/

They only break in one situation: serving brain/ detached from its repo (a subfolder-only deploy), so ../ lands above the root. Inside the repo — where the brain is designed to live — every ref resolves.

✗ Wrong fixstop linking out → forces copying content in → drift returns. Guts the whole point.
instead ↓
✓ Right fixportable publish — an export that rewrites out-refs → absolute URLs (GitHub blob · hosted spec). Still zero copy; now deploys anywhere.

That export is already on the plan as t-0019 (static-publish + a relative-path lint). Nothing to undo today — just a feature to promote when we want a shareable, detached brain.

06-30 · 005Board-first — the canvas is where explanations live›

The decision: when my reply would be a substantial summary or explanation, the board is the reply — it lands here, and the terminal gets one line (📋 … → on the canvas). Not written twice.

We dropped the Stop-hook “nag”. A hook can't tell what's board-worthy — it only sees length, so it over- or under-fires, and it never boards anything itself; it just pokes me, and the call is still mine. So the fix puts that judgment in the natural flow instead of bolting on a dumb detector.

06-30 · 004A scrollable whiteboard — and boarding that doesn't rely on memory›

This is a board — the kind you said my summaries should be. Two things landed this turn.

1 · The whiteboard is real. The canvas is now a white container with its own scroll: a long board scrolls inside it, while the whole page — this box, Earlier, the doors, the footer — scrolls together as one. A scrollable whiteboard, not a trapped panel.

2 · Boarding no longer relies on me remembering. Laying the turn's summary here is now a step of the STAMP — the routine a Stop hook already enforces on every substantive turn. 06-30 · 003Whiteboard, not notebook — and what “stamping” is›

You expected the canvas to show one current thing, not a growing pile — the better instinct. So it's a whiteboard now: the newest board sits full and live up top; earlier ones fold into the Earlier strip below (click any to reopen). The stack caps at five — older boards fall away.

Canvas · workinglive, rich, freeform boards — explanations as they happen. This surface.
promote ↓
Brain · settledcurated log · decisions · learnings · plan — updated by stamping.

Stamping keeps that settled layer current: after a substantial turn I prepend a one-line log row, rewrite the NOW focus, refresh the machine-readable digest, record any decision/gotcha, and flip the plan status. A Stop hook enforces it — change code without stamping and it blocks. The canvas is where thinking is formed; stamping is how the durable parts get kept.

06-30 · 002How Claude Code reaches for the canvas›

The handoff isn't magic — it's a SessionStart hook. On every fresh session it injects the brain digest and a standing instruction about the canvas into Claude Code's context, so by turn one it already knows this surface exists and when to use it.
