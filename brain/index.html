Always publishable — you were right; resolve links at runtime, not build time

07-02 · board 007

Your instinct survived an adversarial review: “brain/ is always publishable” should be an invariant — point any static host at the folder, any moment, no build step. The “portable publish” from board 006 fails that by construction: it is a build step, and its spec snapshots quietly re-open the drift link-never-copy exists to kill. You caught a real flaw.

The measured surface: 158 out-of-brain/ links across 13 pages — 116 (73%) are anchored refs into the two spec docs, 29 point at source, 13 at meta. That 73% is the load-bearing fact: GitHub's blob view doesn't render HTML (source view, dead #anchors), so any “rewrite to GitHub” scheme fails most of the surface. Panel verdicts (one adversarial critic per candidate):

✗ absolute links at write timedead — kills local-first + offline; bakes a mutable URL into append-only records
✗ publish / export build stepdead for this — it is the banned step; snapshot copies reintroduce drift
✓ runtime resolverviable — zero fatal flaws; five serious ones, each with a concrete fix, hardened below

The hardened design: links stay relative on disk (local-first, disk-true). Every page head gets one meta naming the repo's source base (blob/HEAD/ — survives branch renames) plus the expected package name. brain.js probes ../package.json once per load — no caching (same-origin mode flips poisoned a cache), and the name must match (defeats SPA soft-200s and stranger package.jsons). Detached → out-links resolve to the source base at click time (one delegated listener — no flash, no race); no base configured → they dim with a “lives in the repo” tooltip. ~60–90 lines of vanilla JS in the file every page already loads. Zero build. Zero deps.

And the free path works today: deploy the whole public repo (GitHub Pages, or Vercel with root = repo root) — the probe finds package.json, everything stays relative, and the specs render with working anchors. That same deploy later doubles as the rendered base for the 73% when brain/ ships alone. t-0019 reshapes: resolver + a stamp-time link lint now; the snapshot exporter demotes to optional polish.

07-01 · 006The brain links outside itself — on purpose›

The out-links aren't sloppiness — they're the spine. Sandpaper's one rule is link, never copy: the brain is a thin index over canonical truth (the spec, the source), never a second copy of it. Copy the content in and it drifts the moment the code changes.

They only break when brain/ is served detached from its repo (a subfolder-only deploy). Partly superseded by board 007: the fix moved from a publish step to a runtime resolver — the owner pushed back on “publishable only after a build,” and the adversarial panel proved them right.

06-30 · 005Board-first — the canvas is where explanations live›

The decision: when my reply would be a substantial summary or explanation, the board is the reply — it lands here, and the terminal gets one line (📋 … → on the canvas). Not written twice.

We dropped the Stop-hook “nag”. A hook can't tell what's board-worthy — it only sees length, so it over- or under-fires, and it never boards anything itself; it just pokes me, and the call is still mine. So the fix puts that judgment in the natural flow instead of bolting on a dumb detector.
