Gotchas & verdicts

So they're never re-learned. One claim, one why, a back-link to where it bit. · ▲ gotcha · ◆ verdict

◆ Verdict · strategy

The browser is the eyes, not a worse mouth. Don't replicate the terminal's chat on the page — the terminal owns steering; the page's edge is state-made-visible and artifact-made-touchable. pivot →

◆ Verdict · architecture

claude -p is the same Claude, not a subpar one. Capability is a policy dial (--allowedTools), observability is in the stream; only interactive choices are staged. →

▲ Gotcha · CSS cascade

Extracting tokens to theme.css does nothing if the importer keeps its own :root. @import loads first, so a later duplicate :root in brain.css wins — overrides to theme.css silently no-op. The migrate step left it as "harmless redundancy"; it isn't. Drop the consumer's :root entirely. decision →

▲ Gotcha · amux deep-dive

One doc = one folder = one session; the file is the memory. So --resume is an accelerator, not a dependency — a fresh session rebuilds from the file. →

▲ Gotcha · fs.watch

fs.watch can fire with a null filename. Treat null as a match or you silently drop live-reloads (Linux/inotify, macOS under load). fix →

▲ Gotcha · env hygiene

Scrub the child env before spawning claude. An exported ANTHROPIC_API_KEY silently bills the API even on a plan; drop it + the CLAUDECODE markers. →
