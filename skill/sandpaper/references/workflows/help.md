# Help workflow

Show the Sandpaper action set in these groups, then offer to execute one.

## Maintain the brain

- `stamp` — stamp the brain after a substantive turn with the six-step update.
- `log` — add one work-log row, the heartbeat.
- `plan` — add or flip a task or initiative on the plan board.
- `decide` — record a decision, or open or resolve a question.
- `learn` — record a gotcha or verdict learning.
- `canvas` — elevate an explanation into a board on the cover's canvas.
- `sync` — reconcile the brain against the code and heal drift.
- `release` — draft notes from the brain, confirm a semver bump, tag, and push.

## Set up and run

- `init` — scaffold a new brain by harvesting, interviewing, and generating.
- `open` — start the server and open the brain in a browser.
- `serve` — serve the brain or another document with the on-page refine toolbar.
- `theme` — re-skin the whole brain from one brand colour.
- `help` — show this list.

Also note concisely that automatic maintenance uses a session-start hook to rehydrate from the
digest and a stop hook to prompt for a stamp when a turn changed code but not the brain. Do not dump
file contents.
